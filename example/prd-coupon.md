# 用户优惠券系统 PRD

## 1. 背景与目标

为提升用户复购率和活跃度，平台需要上线优惠券功能。用户可以在指定场景领取优惠券，并在下单时使用优惠券抵扣部分金额。

## 2. 功能范围

### 2.1 优惠券类型

系统支持三种优惠券类型：

- **满减券**：订单满足一定金额后可抵扣固定金额（如满100减20）
- **折扣券**：对订单金额按比例折扣（如8折券，最高抵扣50元）
- **无门槛券**：无最低消费要求，直接抵扣固定金额（如5元无门槛券）

### 2.2 优惠券领取

- 用户在活动页面点击"领取"按钮获取优惠券
- 每种优惠券每人限领 N 张（由运营配置，默认1张）
- 优惠券有总库存限制，库存为0时不可领取
- 已领取的优惠券进入用户的"我的优惠券"列表
- 领取时需要校验：用户登录状态、领取次数、库存、优惠券有效期

### 2.3 优惠券使用

- 用户在结算页面选择可用优惠券
- 系统自动筛选当前订单可用的优惠券（根据券类型、门槛金额、适用商品范围）
- 同一订单只能使用一张优惠券
- 使用优惠券后，订单实付金额 = 原价 - 优惠金额
- 若优惠金额大于订单金额，实付金额为0（不可为负数）
- 优惠券使用后状态变为"已使用"

### 2.4 优惠券过期

- 每张优惠券有有效期（开始时间 ~ 结束时间）
- 超过有效期的优惠券状态自动变为"已过期"
- 已过期的优惠券不可使用，在列表中灰显

### 2.5 退款场景

- 若订单使用了优惠券且发生全额退款，优惠券退还给用户（状态恢复为"未使用"）
- 退还的优惠券如果已过期，则不可再次使用
- 部分退款不退还优惠券

### 2.6 后台管理

- 运营人员可在后台创建、编辑、下架优惠券活动
- 可配置：优惠券名称、类型、面额/折扣比例、门槛金额、适用范围、库存数量、有效期、每人限领数量
- 可查看领取和使用统计数据

## 3. 非功能要求

- 领取接口需支持高并发（峰值 5000 QPS）
- 库存扣减需保证原子性，避免超发
- 用户优惠券列表查询响应时间 < 200ms
