# PRD 分析报告：coupon-system

## 概述
本 PRD 描述了一个用户优惠券系统，核心目标是通过优惠券功能提升用户复购率和活跃度。系统涵盖优惠券的创建、领取、使用、过期和退款等完整生命周期管理。

## 功能模块拆解

### 模块 1：优惠券类型管理
- **描述**：系统支持三种优惠券类型（满减券、折扣券、无门槛券），各有不同的计算逻辑
- **涉及角色**：运营人员（配置）、系统（计算）
- **核心流程**：
  1. 运营创建优惠券时选择类型
  2. 系统根据类型执行不同的优惠计算逻辑
- **输入/输出**：
  - 满减券：门槛金额 + 抵扣金额 → 满足门槛则减固定金额
  - 折扣券：折扣比例 + 最高抵扣 → 按比例折扣，不超过上限
  - 无门槛券：抵扣金额 → 直接减固定金额
- **业务规则**：折扣券需有最高抵扣限额；优惠金额不能使实付金额为负数
- **依赖模块**：无

### 模块 2：优惠券领取
- **描述**：用户在活动页面领取优惠券，需校验多重条件
- **涉及角色**：普通用户
- **核心流程**：
  1. 用户进入活动页面
  2. 点击"领取"按钮
  3. 系统校验：登录状态、领取次数、库存、有效期
  4. 校验通过则发券，优惠券进入"我的优惠券"列表
- **输入/输出**：用户ID + 优惠券ID → 领取成功/失败
- **业务规则**：每人限领N张（运营配置，默认1张）；总库存为0时不可领取；需高并发支持（5000 QPS）；库存扣减需原子性
- **依赖模块**：用户登录系统

### 模块 3：优惠券使用
- **描述**：用户在结算页选择可用优惠券抵扣订单金额
- **涉及角色**：普通用户
- **核心流程**：
  1. 用户进入结算页
  2. 系统自动筛选可用优惠券（类型匹配、满足门槛、适用范围）
  3. 用户选择一张优惠券
  4. 系统计算实付金额 = 原价 - 优惠金额（最低为0）
  5. 优惠券状态变为"已使用"
- **输入/输出**：订单信息 + 优惠券 → 实付金额
- **业务规则**：同一订单只能用一张券；优惠金额 > 订单金额时实付为0不可为负；需匹配适用商品范围
- **依赖模块**：订单系统、优惠券类型管理

### 模块 4：优惠券过期
- **描述**：优惠券超过有效期后自动失效
- **涉及角色**：系统（自动处理）
- **核心流程**：
  1. 系统定时检查优惠券有效期
  2. 超过结束时间的优惠券状态变为"已过期"
  3. 已过期优惠券在列表中灰显，不可使用
- **输入/输出**：当前时间 vs 有效期 → 状态变更
- **业务规则**：有效期为开始时间~结束时间的区间
- **依赖模块**：无

### 模块 5：退款场景
- **描述**：使用优惠券的订单退款时，处理优惠券退还逻辑
- **涉及角色**：普通用户、系统
- **核心流程**：
  1. 用户发起退款
  2. 全额退款 → 退还优惠券（状态恢复"未使用"）
  3. 检查退还的优惠券是否已过期
  4. 部分退款 → 不退还优惠券
- **输入/输出**：退款类型 + 优惠券信息 → 是否退还
- **业务规则**：仅全额退款退还；退还后若已过期则不可再使用
- **依赖模块**：订单系统、优惠券过期模块

### 模块 6：后台管理
- **描述**：运营人员通过后台管理优惠券活动
- **涉及角色**：运营人员
- **核心流程**：
  1. 创建优惠券活动（配置各项参数）
  2. 编辑已有活动
  3. 下架活动
  4. 查看领取和使用统计
- **输入/输出**：活动配置参数 → 活动创建/更新
- **业务规则**：可配置项包括名称、类型、面额/折扣、门槛、适用范围、库存、有效期、限领数
- **依赖模块**：优惠券类型管理

## 测试关注点汇总
| 序号 | 功能模块 | 关键测试点 | 复杂度 | 建议优先级 |
|------|---------|-----------|--------|-----------|
| 1 | 优惠券类型管理 | 三种券类型的优惠金额计算逻辑 | 中 | P1 |
| 2 | 优惠券领取 | 多重校验（登录、限领、库存、有效期）+ 并发控制 | 高 | P0 |
| 3 | 优惠券使用 | 可用券筛选逻辑、实付金额计算、状态流转 | 高 | P0 |
| 4 | 优惠券过期 | 定时失效、状态变更、UI展示 | 低 | P2 |
| 5 | 退款场景 | 全额/部分退款的券退还逻辑、过期判断 | 中 | P1 |
| 6 | 后台管理 | CRUD操作、参数校验、统计数据准确性 | 中 | P1 |

## 待确认事项
1. 折扣券的折扣比例范围是多少？（如1折~9.9折？）
2. "适用商品范围"的具体规则是什么？按商品ID、分类、还是店铺维度？
3. 全额退款退还优惠券后，限领次数是否恢复（即用户能否再领一张）？
4. 库存扣减的并发方案是否有指定技术实现（如Redis原子操作）？
5. 优惠券过期的检查机制是定时任务还是查询时实时判断？
