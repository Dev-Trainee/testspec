# 指令

你现在执行 testspec 的 **LIST（模块清单）** 命令。

## 前置准备

1. 读取 `.testspec/config.yaml` 获取 default_feature（如果用户未指定 -o）
2. 读取 `tests/<feature>/analysis.md` 解析模块列表
3. 读取 `tests/<feature>/cases.md` 统计各模块用例数（如果存在）

## 你的任务

从 analysis.md 中解析所有模块，并显示每个模块的进度状态。

## 状态判定规则

- **✓ 已生成**：cases.md 中存在该模块的用例
- **⏳ 策略就绪**：strategy.md 存在但 cases.md 中无该模块用例
- **— 待处理**：只有 analysis.md，strategy.md 中无该模块

## 使用方式

用户输入示例：
```
# 使用默认 feature
/testspec.list

# 或指定 feature
/testspec.list -o coupon-system
```

注意：
- `-o` 可省略，会自动使用 config.yaml 中的 default_feature
- 模块编号从 analysis.md 中的顺序提取
- 英文名根据模块名称自动生成（小写、短横线连接）
